# How-To Index

## Purpose
- Keep short references to proven solutions that are likely to repeat.

## When to Load
- Load during planning or when a known issue appears during execution.
- Do not load by default on cold start.

## How to Use
1. Add one file per recurring problem: `how-to-<topic>.md`.
2. Keep entries deterministic: trigger, exact steps, validation, rollback notes.
3. Link each new how-to file from this index.
